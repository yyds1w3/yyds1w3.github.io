Gdb入门

发表于2025-02-02|更新于2025-02-03
|总字数:528|阅读时长:1分钟|浏览量:|评论数:

开始与退出

首先我们要启动gdb

  • gdb <program><–调试可执行文件
  • gdb <program> --args 参数<–带参数调试
  • q<–退出

断点

其次我们想要查看某一刻程序的变量

断点的设置

  • break 函数名/行数
  • break *地址
  • break 文件:行数
  • break xxx if 条件—条件断点

断点的查看与删除

  • info breakpoints—查看所有断点
  • delete [断点编号]—删除指定断点
  • delete breakpoints—删除所有断点

变量的监视

  • watch [变量名]—变量写入时暂停
  • rwatch [变量名]—变量读入时暂停
  • awatch [变量名]—变量读写时暂停

运行

控制

  • run或r—开始运行
  • start—停在main函数开头
  • continue或c—继续

单步执行

  • next 或 n—执行下一行（跳过函数调用）
  • step 或 s—执行下一行（进入函数内部）
  • stepi (n)—单步二进制命令
  • finish—执行完当前函数并返回
  • until [行号]—执行到指定行号

查看代码和值

查看代码

  • list 或 l
  • list [函数名]

查看变量/寄存器/内存

  • print 或 p 变量—打印出变量的值
  • print *arr@num—打印数组
  • info register—查看寄存器
  • x/[格式] [地址]—查看内存的值

格式–>[count][format][size]
count: 查看的单元数量—>1,2,3…n
format: 显示的形式—>x(十六进制),o(八进制),t(二进制),d(有符号十进制),u(无符号十进制),s(字符串),i(反汇编指令)
size: 单元的大小—>b(1字节),h(2字节),w(4字节),g(8字节)
address: 地址—>具体地址,寄存器,变量名,表达式

查看调用栈

  • backtrace或bt—查看调用栈
  • frame [编号]—切换到指定栈帧
  • up/down—切换到上层/下层栈帧

查看反汇编代码

  • disas(semable)

UI交互

  • display 模式

    命令 功能
    src 源码
    asm 汇编
    split 源码和汇编
    regs 寄存器

Tips: 界面混乱时Ctrl+L刷新界面

其他

  • set [var]=[valuve]—设置变量
  • call [函数名](参数)—调用函数

Tips: 交互模式下Enter键是继续执行上一个命令
